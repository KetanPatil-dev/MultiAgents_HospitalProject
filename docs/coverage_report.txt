 7. ROOT-CAUSE PATTERNS (3 failure modes)
================================================================================

 PATTERN A — INSTANT CRASH ("0 actions, 0.000 s")  — ~20 levels affected
 ---------------------------------------------------------------------------
    Symptom: client exits during setup or first turn. No actions emitted.
    Hits: SAanagram, SAboXboXboX, SACrunch, SADangerBot, SAFirefly, SALazarus,
          SAOptimal, SAsoko3_{05,06,07,08,16,128}, SAsokobanLevel96, SASolo,
          SAsorting, SAtest, SATheRedDot, SAtowersOfHoChiMinh{05,10,26},
          SAtowersOfSaigon{05,5,10,26}, SAWatsOn
    Likely cause: an unhandled level feature in level_parser.py, OR an
    assertion in setup() that fires on certain box/agent configurations
    (e.g., levels with many agents/boxes of the same colour, or where the
    Hungarian assignment receives an unexpected input shape).
    Investigation: run ONE of these (e.g. SAanagram) with stderr captured
    and look at the traceback / last log line before exit.
    Estimated payoff if fixed: 15-20 SA levels unlocked, ~5-10 percentage
    points of total coverage. Possibly some MA levels too.

 PATTERN B — NOOP LOOP / SETUP NEVER PRODUCES A VALID FIRST PLAN  — ~10 levels
 ---------------------------------------------------------------------------
    Symptom: client emits NoOp every turn until hitting MAX_STEPS=19,500.
    Hits: MAPF02C, MAPF03C, MAybetwo, noone, SAsoko3_32, SAsoko3_64,
          (some are mixed with stuck-execution behaviour)
    Likely cause: CBS hits node-cap, HCA* fails, and reactive preplanning
    cannot find an initial plan for any agent. The stuck-recovery nudge
    fires every 15 turns but moves don't help.
    Investigation: look at a single stderr — is it "preplan FAILED" forever
    for every agent? Or does at least one agent have a plan that other
    agents block?
    Estimated payoff if fixed: 5-10 levels unlocked.

 PATTERN C — STUCK MID-EXECUTION  — ~10 levels
 ---------------------------------------------------------------------------
    Symptom: client makes progress (10-300 actions) then deadlocks for the
    rest of the budget. Often paired with the "red + cyan" pattern in
    Thomas Apartment.
    Hits: thomasAppartment_{redcyan,redbluecyan,redcyanpurple,full},
          MADS, DECrunchy, DatzCrazy, TeamAgent, MAPFreorder*,
          MAPFslidingpuzzle, SAlabyrinth, SAthomasAppartment
    Likely cause: agents partially succeed, then enter a swap/escape
    oscillation that the cycle-detector breaks but with no productive
    follow-up. The negotiation refusal is correct, but after a refusal
    there is no Plan B — the agent stays stuck.
    Investigation: trace one (e.g. MADS) and find what happens at the
    moment of "Negotiation cycle detected".
    Estimated payoff if fixed: 5-8 levels unlocked.

 8. PRIORITISED FIXES (highest impact-per-effort first)
================================================================================

  FIX 1 — Diagnose the instant-crash pattern (PATTERN A)              ~2 hours
  --------------------------------------------------------------------------
    Run SAanagram with full stderr capture. If it's a parsing crash,
    fix the parser. If it's an assertion in setup(), guard it or fix
    the underlying assumption. Single highest-leverage fix; could
    unlock 15+ levels in one change.

  FIX 2 — Diagnose red+cyan failure in Thomas Apartment (PATTERN C)   ~3 hours
  --------------------------------------------------------------------------
    All 4 Thomas Apartment failures contain red+cyan together. Trace
    one (redcyan) with stderr capture. Likely a same-corridor
    arbitration issue between agents 1 and 2 in the apartment topology.
    If fixed, 4 levels unlocked.

  FIX 3 — Agent-agent swap support for MAPF (PATTERN B / MAPF reorder) ~6 hours
  --------------------------------------------------------------------------
    Our obstacle-swap layer handles BOX obstacles; pure agent-agent
    swap conflicts (no boxes) don't trigger negotiation. Implement an
    agent-as-obstacle path: when agent A's planned cell is occupied by
    agent B with no plan, force B to take the OPPOSITE move A wants
    (so they pass through each other's old cells). This is a known
    MAPF technique — "agent push" — and unlocks reorder variants and
    SAslidingpuzzle.

  FIX 4 — Productive recovery after negotiation refusal               ~3 hours
  --------------------------------------------------------------------------
    Currently, after "Negotiation cycle detected", the agent goes
    quiet and the level deadlocks. Add a Plan B: assign an escape
    cell so the agent physically moves away from the contested zone,
    allowing the topology to change. Already have _find_escape_cell;
    just wire it into the cycle-refusal path.

 9. LARGER IMPROVEMENTS (post-May 21 / future-work slide for the video)
================================================================================

  IMP 1 — Articulation-point analysis instead of BFS-per-cell
    Current chokepoint detection runs BFS-with-cell-removed for every
    goal cell. Replacing with one Tarjan's articulation-point pass
    over the floor graph would be O(V+E) instead of O(V * (V+E)) and
    catches the same cells. Mostly cosmetic for video, but a credible
    classical-CS reference.

  IMP 2 — Anytime CBS with best-so-far instead of strict success
    Current anytime_cbs_solve returns None if no node cap finds a
    plan. A better version tracks the lowest-cost partial assignment
    seen and returns it when the budget runs out. Couples with a
    runtime layer that handles the un-planned agents reactively.

  IMP 3 — Admissible Joint A* heuristic
    Current Joint A* heuristic sums per-agent BFS distances; this
    over-counts when agents share path segments. A max-of-distances
    variant would be admissible. The level-size gate (≤ 2 agents)
    avoids the problem in practice, but acknowledging it in the video
    pre-empts a question from the graders.

  IMP 4 — Per-level statistics / visualisation pipeline
    negotiations.json already captures the TaskSwap log. A 100-line
    renderer (path overlay + negotiation arrows on the level grid)
    would upgrade the video from "watch the GUI" to "watch the
    agents reason."

  IMP 5 — Replace random.choice nudges with deterministic
    Two paths still call random.choice (obstacle-unblock + stuck nudge).
    Already planned for tomorrow. Replacing with rotated-direction
    selection makes the determinism story complete.

  IMP 6 — Split manager.py into a package
    2616 lines in one file with a 400-line conditional cascade in
    _preplan_for_agent. A clean copy-move into manager/{setup,
    obstacles,chokepoints,hca,joint_astar,conflicts}.py would change
    a reader's perception of architectural quality. No logic change.

  IMP 7 — Box dependency graph + topological scheduling
    Build a DAG of "box X must be placed before box Y because Y's
    delivery path passes through X's goal cell". Topologically order
    deliveries. Generalises the chokepoint defer. New algorithmic
    idea, defensible in the video, but ~1 day of work — only do this
    if Fix 1+2+3+4 are done and there's runway.

 12. POSSIBLE WAYS TO SOLVE THE FAILURES  (per-bucket fix proposals)
================================================================================

  --- BUCKET A1: INSTANT CRASH (2 levels here, ~20 SA levels, total ≈22) ---

  Likely cause: an unhandled level feature crashes during setup —
  most probably either:
    (i)   the level uses a colour name our parser doesn't recognise,
    (ii)  the level has a box-character that has no corresponding goal
          AND the level is structured so the "deco box corridor-
          clearing injection" raises an AssertionError,
    (iii) Hungarian receives an empty / shape-mismatched cost matrix
          for a colour group with no plannable boxes.

  Recommended fix (highest leverage in the whole project, ~2 hours):
      Run MAchoupo2 (smaller than sigMAS) with full stderr capture.
      Tail the last 20 lines before exit; the Python traceback names
      the file + line + exception. Patch the offending assumption.
      Re-run; if MAchoupo2 solves, re-run all 22 instant-crash levels.

  Estimated payoff: 0-22 levels unlocked depending on whether the
  bug is shared (likely shared based on uniform "0 actions" signature).


  --- BUCKET B1: MAX_STEPS NOOP LOOP (5 levels) ---

  Likely cause: every agent's individual A* fails (CBS excludes them
  all), HCA* also fails, and reactive preplanning detects only AGENT
  obstacles (not box obstacles), which our negotiation layer doesn't
  resolve. The stuck-recovery nudge bumps an agent every 15 turns but
  the global topology never opens up.

  Recommended fix #1 — Agent-as-obstacle negotiation (~6 hours):
      When agent A's A* fails because agent B occupies the only path,
      route through a new mechanism analogous to box-obstacle swap:
          (a) detect agent-as-obstacle in _find_obstacles
          (b) if obstacle is agent B with NO plan: command B to take
              a move IN the same direction A wants (so B "yields by
              continuing", clearing the cell after one turn)
          (c) if B has a plan: A waits one turn with a (cell, t)
              constraint and tries again
      This is the MAPF "agent push" pattern, well-known in literature.

  Recommended fix #2 — Bigger anytime CBS budget for these levels (~30 min):
      Current cap schedule is (50, 200, 800, 3000) under a 5 s budget.
      The competition allows 180 s. Allow a much larger schedule
      (50, 500, 5000, 30000) under a 30 s setup budget. Pure runtime
      change, no algorithmic risk. Likely unlocks 1-3 of these levels.

  Estimated combined payoff: 3-5 levels unlocked.


  --- BUCKET C1: STUCK MID-EXECUTION (34 levels — the big bucket) ---

  Likely causes (multiple contributing):
    (i)   Negotiation cycle detector refuses a swap then has no Plan B.
          Agents go idle. Already on the fix list (§8 Fix 4).
    (ii)  Box-blocking-box chokepoint not detected because the BFS-
          with-cell-removed analysis is run ONCE at setup; subsequent
          state changes can create new chokepoints we never see.
    (iii) Joint A* is gated to ≤2 agents AND ≤450 cells; many of these
          levels are 3-5 agents AND ≤450 cells, where Joint A* would
          actually work but isn't tried.
    (iv)  HCA*'s rigid priority order (low-id first) is bad for some
          topologies; the alternative-priority rotation in §3 of
          MANAGER_FLOW.md exists in spirit but the time budget on
          reordering is too tight.

  Recommended fix #3 — Plan-B after negotiation refusal (~3 hours):
      When _recent_swap_count refuses, instead of going idle:
          - Find an escape cell ≥6 steps from the contested zone
          - Set it as the agent's temp goal
          - Use the existing _find_escape_cell (already in code)
      Already mentioned in §8. Single highest priority for this bucket.

  Recommended fix #4 — Re-run chokepoint analysis mid-game (~4 hours):
      After every K turns OR after every box delivery, re-execute
      _compute_chokepoints on the CURRENT state. Newly-created
      chokepoints get deferred. Modest extra compute (~50 ms/call on
      WardRush-sized levels).

  Recommended fix #5 — Extend Joint A* gating (~1 hour):
      Currently  num_agents <= 2  AND  n_cells <= 450.
      Try         num_agents <= 3  AND  n_cells <= 300
      OR          num_agents <= 4  AND  n_cells <= 150
      Joint A*'s 500k-state cap on 3-agent levels is tractable on
      small grids. Could unlock several small-but-tricky levels in
      this bucket.

  Recommended fix #6 — Time-budgeted priority reorder (~3 hours):
      HCA* currently runs once with priority = agent_id order. After
      the initial pipeline, if any agent failed and 30 s of wall-clock
      budget remains, rotate priority orders (shifts of N, and "failed
      agents first") and re-run HCA* on the rotation. Keep the best.

  Estimated combined payoff for Bucket C1: 8-15 levels unlocked.
